## 三、不合理建议的分析

### ❌ 建议1：扫描上传整本书

**不做的理由**：明确侵犯版权。Usborne 是国际出版社，有版权保护团队。扫描整本书（包括图片）上传到网站，无论是公开还是半公开，都构成对复制权和信息网络传播权的侵犯。

### ❌ 建议2：换名不换对话

**不做的理由**：法律上属于"演绎作品"（derivative work），即在原作基础上做了修改但没有获得原作授权。制作演绎作品的权利属于原作者。

**判断标准**：如果一个熟悉原书的读者看了你的版本，能认出"这就是那本书改的"——那就侵权了。

**安全做法**：只借故事概念（idea），对话和情节必须自己写。最推荐的方法是让孩子读原书后用自己的话复述故事，记录下来就是原创剧本。

### ❌ 建议12：线下演出视频 → 语音分离 → 评分

**不做的理由**：
1. **语音分离质量差**：演出视频中有背景噪音、多人同时说话、背景音乐，提取出来的语音质量远不如直接录音
2. **评分不准确**：基于劣质语音评分没有意义，孩子会得到不公平的低分
3. **技术复杂度高**：需要集成 Whisper 等大模型做语音分离，免费服务器跑不动
4. **替代方案更好**：直接让孩子用手机/电脑录音上传评分，简单、准确、已实现

## 四、部分合理建议的分析

### ⚠️ 建议10：故事推荐

**保留部分**：管理员（Amy/Tom）上传故事 ✅
**暂不做的部分**：小朋友推荐故事得奖励 ⏸

**暂不做推荐的理由**：
1. 运营负担重：需要有大人审核推荐的故事、改编成双语剧本
2. 标准难定义：什么样的故事算"合格"？谁来评判？
3. 奖励机制复杂：怎么发奖励？积分系统？实物？
4. 用户量小：目前就几个小朋友用，推荐功能优先级很低

**替代建议**：Amy 和 Tom 自己选故事。如果后续用户量大了（50+人），再考虑加推荐功能。可以先做一个简单的"我想演什么"投票。

### ⚠️ 建议11：短视频

**保留部分**：互动视频播放器（网页内播放）✅、音频下载 ✅
**暂不做的部分**：真实视频文件下载 ⏸

**暂不做视频下载的理由**：
1. PythonAnywhere 免费版存储空间有限（512MB），视频文件较大
2. 生成视频需要 moviepy + ffmpeg，免费服务器处理慢
3. 音频下载已能满足离线练习需求

## 六、待开发功能 SPEC

### 功能1：注册登录系统 ⭐⭐⭐（P0 最高优先级）

**需求描述**：
- 未登录用户可以浏览剧本，但按钮（认领、练习、录音、下载）不可点击
- 注册后可以使用全部功能
- Amy 和 Tom 为管理员，拥有额外权限

**用户角色**：

| 角色 | 权限 |
|------|------|
| 游客（未登录） | 浏览剧本、切换语言 |
| 注册用户 | 认领角色、练习、录音、评分、下载 |
| 管理员（Amy/Tom） | 以上全部 + 上传新故事 + 管理用户 |

**实现方案**：

| 步骤 | 技术方案 |
|------|---------|
| 1. 用户表 | SQLite 新增 users 表（id, username, password_hash, role, created_at） |
| 2. 注册API | POST /api/register，接收 username 和 password，密码用 werkzeug.security 的 generate_password_hash 加密 |
| 3. 登录API | POST /api/login，用 check_password_hash 验证，成功后存入 flask session |
| 4. 登出API | POST /api/logout，清除 session |
| 5. 当前用户 | GET /api/me，返回当前登录用户信息或 null |
| 6. 权限控制 | 前端：未登录时按钮灰色+提示"请先登录"；后端：API检查session |
| 7. 管理员 | role 字段区分 admin/user，初始化时自动创建 amy 和 tom 两个管理员账号 |

**预估工作量**：2-3天

### 功能2：互动视频播放器 ⭐⭐⭐（P0 最高优先级）

**需求描述**：
- 每个场景做成"类视频"的互动播放器
- 自动播放台词：逐句显示字幕 + 播放标准语音
- 静音模式：关掉语音，小朋友看字幕自己读
- 可暂停、可重播、可切换英文/中文版

**播放器界面设计**：

```
┌──────────────────────────────────┐
│  🦕 The Dinosaurs Next Door       │
│  Scene 1: The New Neighbor        │
│                                   │
│         🎩 Mr. Puff              │  ← 当前说话的角色（大字高亮）
│                                   │
│  "Hello there! I'm Mr. Puff,     │  ← 字幕（大字）
│   your new neighbor."             │
│                                   │
│  🇬🇧 EN  🇨🇳 ZH  🔊/🔇  🎤     │  ← 语言切换/静音/录音
│  ▶ ⏸ ⏮ ⏭                       │  ← 播放控制
│  ═══════════●═══════════════      │  ← 进度（第3句/共9句）
└──────────────────────────────────┘
```

**播放逻辑**：
- 点播放后，按顺序逐句显示台词
- 每句停留3-5秒（根据文本长度自动调整）
- 语音：用浏览器 SpeechSynthesis API 朗读当前台词
- 静音模式：不播放语音，孩子看字幕自己读
- 点🎤录音：录制孩子的朗读，录完自动播放标准语音对比

**标准语音实现选择**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| A. 管理员录音 | 音质好，真人发音 | 每句都录，工作量大 |
| B. 浏览器TTS | 免费，不用录，一键生成 | 音质一般，部分浏览器不支持中文 |
| C. AI语音合成API | 音质好，自动生成 | 需要付费API |

**建议**：先用方案B（浏览器TTS）快速上线，后续再换成管理员录音或AI合成。

**预估工作量**：3-5天
